# エックスサーバー 自動デプロイ設定ガイド

`main` ブランチに push すると、GitHub Actions が自動で
「ビルド → エックスサーバーへFTPS転送」を行います。**FileZilla は不要になります。**

そのために、エックスサーバーのFTP情報を一度だけ GitHub に登録します。
（パスワードは暗号化された Secrets に保存されるので、コードには一切残りません）

---

## 1. エックスサーバーのFTP情報を確認する

いつも FileZilla に入れている情報です。エックスサーバーの
**サーバーパネル → 「FTPアカウント設定」** で確認できます。

| 必要な値 | 例 | 説明 |
|---|---|---|
| FTPサーバー（ホスト名） | `sv1234.xserver.jp` | FileZilla の「ホスト」 |
| FTPユーザー名 | `xserver-account` | FileZilla の「ユーザー名」 |
| FTPパスワード | （ご自身のパスワード） | FileZilla の「パスワード」 |
| アップロード先フォルダ | `/taikisato.com/public_html/` | FileZilla でアップしている先のパス |

> アップロード先フォルダは、FileZilla で接続したときに
> **dist の中身（index.html など）を置いている階層**です。
> エックスサーバーでは通常 `/（ドメイン名）/public_html/` になります。
> **末尾のスラッシュ `/` まで含めて**控えてください。

---

## 2. GitHub に Secrets を登録する

1. リポジトリ https://github.com/INxST/st-portfolio を開く
2. **Settings**（上部タブ）→ 左メニュー **Secrets and variables → Actions**
3. **New repository secret** ボタンから、以下の4つを1つずつ登録する

| Name（この通りに入力） | Secret（値） |
|---|---|
| `FTP_SERVER` | ホスト名（例 `sv1234.xserver.jp`） |
| `FTP_USERNAME` | FTPユーザー名 |
| `FTP_PASSWORD` | FTPパスワード |
| `FTP_SERVER_DIR` | アップロード先フォルダ（例 `/taikisato.com/public_html/`） |

> Name は**大文字・スペルを正確に**。値の前後に余計な空白を入れないこと。

---

## 3. 動作確認

1. リポジトリの **Actions** タブを開く
2. 左の **「Deploy to Xserver (FTP)」** を選び、右の **Run workflow** で手動実行
3. 緑のチェックが付けば成功。taikisato.com に反映される
4. 以降は **main に push するだけ**で自動的に反映される

---

## うまくいかないとき

- **認証エラー（530 など）** → ユーザー名／パスワードの打ち間違い。Secrets を登録し直す
- **接続できない** → `protocol: ftps` / `port: 21` で繋がらない場合、エックスサーバー側で
  FTPS が有効か確認。どうしても繋がらなければ `.github/workflows/deploy-xserver.yml` の
  `protocol` を `ftp` に変更（ただし暗号化されないため非推奨）
- **ファイルは上がるが表示が変** → `FTP_SERVER_DIR` の階層が違う可能性。
  FileZilla で実際にアップしている階層と一致しているか確認
